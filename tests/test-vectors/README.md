# qw3 test vectors

These are early local/reference regression vectors for `qw3`.

They are not official Qwen vectors yet. Their purpose is the same narrow
guardrail used by `ds4`: catch tokenizer/template, top-logit, generation, and
session-cache regressions before they turn into vague quality failures.

Current vector source:

- Chat template/tokenization: matched against `llama-tokenize`.
- Top logits/generation/session payload: captured from the current CPU
  reference path after the ChatML template fix.
- `ciao-logits-v1.json`: generated by `--dump-logprobs` with `-n 2`,
  `--logprobs-top-k 5`, `--ctx 128`, greedy sampling.
- `--dump-trace`: smoke-checked for schema, embedding event, final layer event
  and top-1 logit. Full trace files are meant for local CPU/Metal comparison
  and are not stored as fixtures yet.

Run:

```sh
make test-vectors
```

Metal smoke coverage is intentionally separate because it requires a visible
Apple Metal device. It checks the Metal RMSNorm diagnostic, CPU-vs-Metal decode
for `ciao`, persistent Metal session allocation/clear, session-buffer
embedding into `x0`, session-buffer RMSNorm into `x1`, session-buffer QKV
projection into scratch, session-buffer Z/gate projection after QKV,
session-buffer DeltaNet conv1d, session-buffer Q/K L2Norm, session-buffer SSM
gates, session-buffer zero-state DeltaNet recurrence, session-buffer gated
RMSNorm, session-buffer DeltaNet `ssm_out`, session-buffer GQA projection/cache,
session-buffer GQA single-token `attn_o`, session-buffer GQA cached attention,
complete Metal layer/MoE, and exact two-token plain Metal generation (`Here's`):

```sh
make test-metal-smoke
```
